Collection

collection下主要有list及set接口，collection是一个高度抽象出来的集合。包含了集合的基本操作，增、删、清空、遍历、大小、是否为空等等方法。

且collection的所有子类都必须实现两种构造方法，无参和参数为collection的构造方法。后者可以用来转换collection的类型。

List

list是一个继承自collection的接口。list是有序的！队列,list中的每一个元素都有一个索引，第一个索引是0，之后依此+1，相较set而言，list允许有重复元素。既然list继承自collection，它包含collection中所有的方法。此外，由于是有序队列，它包含了对指定位置上的元素的一些增、删、查、改、截取子队列的操作。

Set

set的api与collection一致。set没有重复的元素。

AbstractCollection

abstractCollection是一个抽象类，它实现了collection中大部分的方法，除size()及iterator()之外。由于其实现了collection的大部分方法，所以其方便了其他类实现collection接口。
